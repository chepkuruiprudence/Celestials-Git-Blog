I. What is Git

If you have ever worked on a project whether it's writing a paper, developing a piece of software or graphics design you have often met yourself with multiple versions of your work saving files like finalWork.docx or ISwearThisIsFinal.docx For you and all other persons doing projects as such, Git is a powerful tool that easens the burden.

Git is a Version Control System (VCS) ; a tool that helps you manages changes to files over time. With Git, you get to know what changed, when, why and by who was the change done. Whether working alone or with a team Git allows for easy collaboration and it allows you to revert to previous versions of your project.

A version control system is like a database of file changes that enable you to:

  • Track changes to your project and store versions of the files
  • Collaborate with team members without overwriting others work
  • Review and revert to previous versions of your work
  • Experiment new feature without breaking the main project
There are other Version Control systems like Perforce, Subversion and Mercurial but Git is the most widely used and has become more popular.

In this blog, we'll focus on Git and how it works as well as how it integrates with GitHub, a platform built to hold Git repositories,(folders/directories), online.

Git is the tool for use to track changes to your project; whether you are a designer, developer, writer, project manager, working solo or with a team, Git will help you collaborate and share your work without breaking the project or encountering Chaos in file naming and renaming.

II. Getting Started with Git

To get started with Git, you need to install it on your local machine.

1️⃣ Check whether Git is Installed

Open your Terminal (Command Prompt on Windows, Terminal on macOS/Linux) and type the following command: git --version if Git is installed you'll see an output with a Git Version number like this: if you get an error like Command not found or 'git' is not recognized as an internal or external command it means Git is not installed yet.

2️⃣ Installing Git

For Windows:

  1. Go to https://git-scm.com/downloads
  2. Click on the Windows download link.
  3. Run the installer and follow the default prompts (you can safely leave the options as they are unless you have specific preferences).
  4. After installation, reopen Command Prompt and run git --version again to verify it's working.

For macOS:

The easiest way is to install Git via Homebrew. If you already have Homebrew installed, run: brew install git Alternatively, installing Xcode Command Line Tools also provides Git: Run: xcode-select --install

For Linux (Debian/Ubuntu-based systems):

Run the following in your terminal: sudo apt update sudo apt install git Once Installed, Verify by running git --version

IV. Initializing a Git Repository

With Git installed and configured let's dive into repositories the core of how Git functions

What is a Git Repository

A Git repository (or repo for short) is a storage space where Git tracks and manages all the changes to your project’s files. Think of it as a digital ledger that records the history of your project, allowing you to view, revert, or branch off into different versions at any point. Let's see how we can create a Git repository:

V. Adding files to the Staging Area

What is the Staging Area

In Git, the staging area is an intermediate space where you can prepare changes before saving/uploading/committing them to the repository. Think of it as a rough draft space where you can review and format your changes.

Tracked and Untracked files

  • Tracked files are files that Git is aware of and monitors for changes.
  • Untracked files are new files in your working directory that Git isn't tracking yet.

Adding files to the Staging Area

To add files to the staging area, use this command (replace 'file-name' with actual name of your folder) git add file-name

Here is how it looks like: To add multiple files to the Staging area: git add file-1-name file-2-name file-3-name
To add all changes (new, modified, and deleted files) run: git add .

VI. Committing Changes

What is a commit?

A commit in Git is a snapshot of your project's current state. It records changes to the repository and includes a commit message describing the changes.

Writing Good Commit Messages

Good commit messages are crucial for understanding the history of your project. The commit messages should be a continuation of the statement "This commit is going to". In summary:

  • Be concise and descriptive:Summarize what changes were made and why.
  • Use the imperative mood:For example, "Fix bug in user login" instead of "Fixed bug in user login."
  • Include relevant issue numbers:If applicable, reference related issues or tickets.
use git commit -m "commit-message" to commit your changes with the relevant commit message
For Example:
Run git commit to open a text editor and write a more descriptive commit message if need be.

Viewing Past Commits

To view the commit history: run git log

Reverting to a Previous Commit

To revert your repository to a previous commit: use git revert 'commit-id' This command creates a new commit that undoes the changes made in the specified commit.

VII. Branches

What is a Branch?

A branch in Git is a separate line of development. It allows you to work on different features or fixes independently without affecting the main codebase.

Why Use Branches?

  • Collaboration:Multiple developers can work on different branches simultaneously.
  • Isolation:Work on new features or bug fixes without impacting the main branch.
  • Experimentation:Test new ideas safely before merging them into the main branch.
